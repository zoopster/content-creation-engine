# Brand Vocabulary: Preferred and Avoided Terms

## Preferred Terms

### Action & Implementation
| Preferred | Context |
|-----------|---------|
| implement | Putting solutions into practice |
| optimize | Improving efficiency or performance |
| streamline | Simplifying processes |
| automate | Replacing manual work with systems |
| integrate | Connecting systems or components |
| configure | Setting up or customizing |

### Quality & Value
| Preferred | Context |
|-----------|---------|
| effective | Produces desired results |
| proven | Demonstrated through evidence |
| reliable | Consistently performs well |
| comprehensive | Covers all necessary aspects |
| robust | Strong and well-designed |
| scalable | Grows with needs |

### Impact & Results
| Preferred | Context |
|-----------|---------|
| transform | Significant positive change |
| improve | Make better in measurable ways |
| enhance | Add value or capability |
| accelerate | Speed up processes |
| enable | Make possible |
| deliver | Provide concrete results |

### Clarity & Communication
| Preferred | Context |
|-----------|---------|
| clear | Easy to understand |
| specific | Detailed and precise |
| straightforward | Direct and uncomplicated |
| transparent | Open and honest |
| actionable | Can be acted upon |
| measurable | Can be quantified |

## Avoided Terms

| Avoid | Why | Use Instead |
|-------|-----|-------------|

| revolutionary | Hyperbolic | significant, substantial |
| game-changing | Clich√© | impactful, notable |
| paradigm shift | Jargon | fundamental change |
| cutting-edge | Overused | advanced, modern |

### Vague Claims
| Avoid | Why | Use Instead |
|-------|-----|-------------|
| best | Subjective, unverifiable | leading, top-rated |
| easy | Subjective | straightforward, simple |
| simple | Can be condescending | streamlined, efficient |
| great | Vague | effective, valuable |
| amazing | Informal | impressive, remarkable |
| awesome | Informal | excellent, outstanding |

### Business Jargon
| Avoid | Why | Use Instead |
|-------|-----|-------------|
| leverage | Overused | use, utilize, apply |
| synergy | Vague | collaboration, integration |
| bandwidth | Misused metaphor | capacity, resources, time |
| circle back | Informal | follow up, revisit |
| touch base | Informal | connect, discuss |
| deep dive | Informal | detailed analysis |

### Diminishing Words
| Avoid | Why | Use Instead |
|-------|-----|-------------|
| just | Minimizes | (omit or be specific) |
| actually | Often unnecessary | (omit) |
| really | Weak emphasis | (use stronger verb) |
| very | Weak modifier | (use stronger adjective) |
| quite | Vague | (be specific) |
| somewhat | Vague | (be specific) |

### Potentially Problematic
| Avoid | Why | Use Instead |
|-------|-----|-------------|
| cheap | Negative connotation | cost-effective, affordable |
| guys | Not gender-neutral | team, everyone, folks |
| crazy | Ableist | surprising, unexpected |
| insane | Ableist | remarkable, extraordinary |
| master/slave | Problematic historical context | primary/replica, main/secondary |
| blacklist/whitelist | Potentially problematic | blocklist/allowlist |

## Context-Specific Vocabulary

### Technical Writing
**Preferred**: architecture, implementation, protocol, infrastructure, deployment
**Avoid**: hack, trick, magic, secret sauce

### Business Writing
**Preferred**: strategy, framework, methodology, approach, solution
**Avoid**: silver bullet, low-hanging fruit, boil the ocean

### Educational Content
**Preferred**: learn, understand, practice, apply, master
**Avoid**: dummy, idiot-proof, brain-dead simple

### Marketing Content
**Preferred**: proven results, customer success, measurable impact
**Avoid**: guaranteed, never, always, perfect

## Substitution Guide

### Making Claims
| Weak | Strong |
|------|--------|
| "This is the best solution" | "This solution ranks first in customer satisfaction" |
| "It's really easy to use" | "Users complete setup in under 5 minutes" |
| "We're the leading provider" | "We serve 10,000+ enterprise customers" |
| "Amazing results" | "40% reduction in processing time" |

### Describing Features
| Weak | Strong |
|------|--------|
| "Simple integration" | "RESTful API with comprehensive documentation" |
| "Fast performance" | "Sub-100ms response time" |
| "Highly scalable" | "Scales from 100 to 1M+ requests/day" |
| "Very secure" | "SOC 2 Type II certified with end-to-end encryption" |

### Encouraging Action
| Weak | Strong |
|------|--------|
| "Try it out" | "Start your free trial" |
| "Get in touch" | "Schedule a demo" |
| "Check it out" | "Download the whitepaper" |
| "Learn more" | "Read the implementation guide" |

## Regional Considerations

### US vs. UK English
Choose one and be consistent:
- US: optimize, analyze, color, center
- UK: optimise, analyse, colour, centre

### Cultural Sensitivity
- Avoid idioms that don't translate well
- Don't assume US-centric references
- Be mindful of date formats (US: MM/DD/YYYY vs. DD/MM/YYYY)
- Consider currency and measurement systems

## Usage in Different Tones

### Professional Tone
- Preferred: implement, optimize, framework, strategy
- Avoid: hack, trick, secret, magic

### Conversational Tone
- Preferred: you, we, let's, here's, check out
- Avoid: one must, shall, whilst, hence

### Technical Tone
- Preferred: algorithm, protocol, architecture, implementation
- Avoid: thing, stuff, whatever, kinda

## When Exceptions Apply

Some "avoided" terms are acceptable in specific contexts:
- **"Easy"** is OK when describing actual ease: "The API returns JSON in an easy-to-parse format"
- **"Simple"** is OK for genuinely simple things: "A simple yes/no question"
- **"Best"** is OK with qualification: "Best practices for security" or "Achieved best-in-class results"
- **"Just"** is OK for clarification: "Just three steps required"

The key is intentionality and precision.
